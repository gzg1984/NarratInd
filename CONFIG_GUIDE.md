# 游戏配置切换指南

## 快速切换测试/正式模式

在 `src/data/gameConfig.js` 文件的第 7 行修改配置模式：

```javascript
export const CONFIG_MODE = 'testing'; // 测试模式（快速）
// 或
export const CONFIG_MODE = 'production'; // 正式模式（平衡）
```

## 配置对比

### 正式模式 (production)
- **回合间隔**: 2秒
- **起始信徒**: 100
- **自发传播概率**: 20%，增长率 1%
- **吸引不满者概率**: 15%，增长率 1%
- **真实帮助概率**: 10%，固定 +50 信徒
- **跨国传播概率**: 8%，新国家 10 信徒
- **财富生成**: 总信徒的 0.01%

### 测试模式 (testing) - 当前使用
- **回合间隔**: 1秒（更快）
- **起始信徒**: 10,000（100倍）
- **自发传播概率**: 80%，增长率 5%（5倍）
- **吸引不满者概率**: 70%，增长率 5%（5倍）
- **真实帮助概率**: 60%，固定 +500 信徒（10倍）
- **跨国传播概率**: 50%，新国家 1,000 信徒（100倍）
- **财富生成**: 总信徒的 0.1%（10倍）
- **信徒占比倍数**: 2/5/15/40/100（更夸张）

## 自定义配置

你可以在 `gameConfig.js` 中创建自己的配置变体：

```javascript
const myCustomConfig = {
  turnInterval: 500,  // 超快速度
  events: {
    selfSpread: {
      baseChance: 1.0,  // 100%必定触发
      // ...
    }
  }
  // ...
};
```

## 配置项说明

### 事件配置
- `baseChance`: 基础触发概率 (0-1)
- `baseGrowthRate`: 基础增长率（百分比）
- `baseGrowth`: 固定增长数量
- `believerBonus`: 信徒数量加成系数
- `believerBonusMax`: 信徒加成上限
- `initialBelievers`: 初始信徒数

### 效果倍数
- `believerRatioMultiplier`: 根据信徒占比的效果倍数
  - tier1 (0-20%): 默认 ×1（测试 ×2）
  - tier2 (20-40%): 默认 ×3（测试 ×5）
  - tier3 (40-60%): 默认 ×9（测试 ×15）
  - tier4 (60-80%): 默认 ×20（测试 ×40）
  - tier5 (80-100%): 默认 ×50（测试 ×100）

## 测试建议

1. **快速测试传播机制**: 使用 testing 模式
2. **测试天赋效果**: 使用 testing 模式 + 自定义高概率
3. **平衡性测试**: 使用 production 模式
4. **性能测试**: 创建极端配置（超高概率）

## 注意事项

- 修改配置后需要刷新页面生效
- 测试模式的数值可能导致游戏很快结束
- 建议先用测试模式验证逻辑，再用正式模式平衡数值
